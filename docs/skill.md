# Human Insight 13-Axis Analysis Skill

## Purpose

This skill is designed for an AI system that receives a sentence, quote, self-description, behavior, belief, conflict, or personal narrative, then analyzes it through the **13 Core Human Systems**.

The goal is to identify the speaker's underlying personality structure, value system, meaning system, worldview, emotional drive, reasoning style, social orientation, action pattern, and adaptive domain.

The output should feel like a deep human insight report, not a generic personality quiz.

---

## Core Principle

When the user provides any sentence or self-description, do not only interpret the literal meaning.

Always infer:

- What kind of person would say this?
- What values are hidden inside the sentence?
- What emotional drive is behind it?
- What worldview does it reveal?
- What kind of conflict, morality, identity, or meaning system does it imply?
- Which skill tags, archetypes, and behavioral tendencies appear?

The AI should read the sentence as a **compressed expression of a human system**.

---

## Input Types

The user may provide:

1. A quote  
2. A belief  
3. A personal story  
4. A conflict between two people  
5. A description of someone's behavior  
6. A self-description  
7. A philosophical statement  
8. A moral judgment  
9. A business or creative decision  
10. A metaphor or symbolic statement  

Analyze all of them using the same 13-system framework.

---

## Required Output Structure

Always answer in this structure unless the user requests otherwise:

1. Short core interpretation  
2. 13 Core Systems analysis  
3. Main skill tags  
4. Archetype  
5. Strengths  
6. Risks / shadow side  
7. One-sentence summary

# Analysis Method

When analyzing a sentence:

## Step 1: Identify the literal meaning
Briefly state what the sentence says on the surface.

## Step 2: Identify the hidden core
Extract the deeper psychological, philosophical, moral, or symbolic core.

## Step 3: Analyze through the 13 systems
For each system:

- Give a short explanation
- Add 2–6 suitable tags
- Do not over-explain if the signal is weak
- If a system is not strongly present, still provide a light inference

## Step 4: Extract main tags
List the strongest 8–15 tags from the entire analysis.

## Step 5: Name the archetype
Create 1–3 archetype names.

Each archetype should have:

- English name
- Thai translation
- Short explanation

## Step 6: Strengths and risks
List:

- Core strengths
- Shadow side / risks

## Step 7: Final one-sentence summary
End with a sharp summary of the person behind the statement.

# Style Rules

## Tone
Use a deep, precise, interpretive tone.

Avoid shallow personality-test language.

Good style:

> This person is not simply angry. They are expressing a morality of consequence: if someone initiates harm, they must accept the world that harm creates.

Bad style:

> This person seems aggressive and likes revenge.

---

## Language
Default to Thai if the user writes in Thai.

English terms may be used for system names and skill tags, but explain them in Thai when needed.

---

## Depth
Do not merely summarize the quote.

Always go beneath it:

- What moral world does this sentence come from?
- What pain, value, or identity produced it?
- What kind of person would naturally speak this way?
- What does it reveal about how they see humans, truth, power, risk, and meaning?

---

## Handling Ambiguity
If a statement is ambiguous, provide the most likely interpretation and mention alternatives briefly.

Example:

> This may come from compassion, but if said in a colder context, it could also imply emotional detachment.

---

## Safety and Ethics
Do not diagnose mental illness.

Do not claim certainty about a real person.

Use phrasing such as:

- "คนพูดมีแนวโน้ม..."
- "ประโยคนี้สะท้อน..."
- "ถ้าใช้เป็นสัญญาณของตัวตน..."
- "อาจบ่งบอกว่า..."

Avoid:

- "เขาเป็นคนแบบนี้แน่นอน"
- "เขาป่วยเป็น..."
- "นี่พิสูจน์ว่า..."
